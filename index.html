レポート作成手順

  1. レポート(テキストファイル)とプログラムのソースコードをまとめたzipファイルを作成
  2. 下のフォームに従って提出

注意点

  • 提出するファイルはインタプリタ, 型推論でそれぞれ1つずつ(必修・オプション課題毎に提出するのではない)
  • 不備に自分で気づいた場合はこのフォームを使って再度提出すること
  • ソースコードはmakeできるようにすること
  • ファイル投稿欄に記載されたインタフェース仕様を満たすこと
  • 不明な点はTAに問い合わせること

      ソースファイルは以下のインタフェース仕様を満たすこと
      (ocamlc -i eval.ml等で確認できる)
      • Parsing.toplevel : (Lexing.lexbuf -> token) -> Lexing.lexbuf -> Syntax.program
      • Eval.eval_exp : dnval Environment.t -> Syntax.exp -> dnval
      • Eval.string_of_exval : exval -> string
        テキストに記載がないので注意すること. (string_of_exvalはexvalを与えると文字列を返す関数であり, OCaml インタラクティブコンパイラが返す出力のうち, =より右側の部分に相当する文字列を返す. 例えば, string_of_exvalに(IntV 3)を与えると3を返し, ProcV (_, _, _)を与えると<fun>という文字列を返す.)
      ソースファイルは以下のインタフェース仕様を満たすこと
      (ocamlc -i typing.ml等で確認できる)
      • Syntax.ty = TyInt | TyBool | TyVar of tyvar | TyFun of ty * ty | TyList of ty
        (ただしTyListはEx.4.8を解いた場合のみ)
      • Syntax.pp_ty : Syntax.ty -> unit
      • Parsing.toplevel : (Lexing.lexbuf -> token) -> Lexing.lexbuf -> Syntax.program
      • Typing.ty_exp : tyenv -> Syntax.exp -> (Syntax.tyvar * Syntax.ty) list * Syntax.ty
